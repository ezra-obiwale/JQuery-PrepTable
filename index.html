JQuery-Table

A JQuery function to load data into the table and to search, order, and paginate the data.

JQuery Table

This was created as an alternative to the JQuery DataTable because of the excessive errors gotten from there. This library is a lot easier and cleaner to use.

Usage

Add the following to your web page as shown below

<!DOCTYPE html>
<html>
    <head>
        <link type="text/css" rel="stylesheet" href="/path/to/jquery-table.min.css" />
    </head>
    <body>
        <table>
            <thead> <!-- not required -->
                <tr>
                    <th>First Name</th>
                    <th>Last Name</th>
                    <th>Gender</th>
                </tr>
            </thead>
        </table>
        <script src="/path/to/jquery.js"></script>
        <script src="/path/to/jquery-table.min.js"></script>
        <script>
            $('table').table({
                data: [
                    ['Ezra','Obiwale','Male'],
                    ['Annie','DiMaria','Female'],
                    ['Sandy','Stone','Female'],
                ]
            });
        </script>
    </body>
</html>

Configuration

var config = {
        columns: [],
        data: [],
        ajax: null,
        append: false,
        limits: {values: [10, 20, 50, 100, 200], all: true, default: 10},
        searchable: true,
        label: true,
        pagination: {
            firstLast: false,
            nextPrevious: true,
            numbers: false,
            maxButtons: 12
        },
        loading_message: 'Loading table...',
        empty_message: 'NO ENTRIES FOUND.',
        display_errors: false,
        log_errors: true,
        log_data: false,
        alert_errors: true,
        createRow: function (row, data, index) {}
    };
$('table').table(config);
property value default description
ajax string|object null A string path to the location of the data or an object with 2 string properties - url and type, which may be 'get' or 'post'.
alert_errors boolean true Indicates whether to alert any errors in operation.
append boolean false Indicates whether to append new rows to existing ones.
columns array empty array An OPTIONAL array of objects containing information about the property to map to the <td> in each data row. Each object must have a string data property with the data object property as the value. This is only used by the default createRow function. Optionally, each object may also have boolean sortable and searchable properties indicating where the column should be sortable and searchable respectively.
createRow function function The function to call to create each row data. Three parameters are passed into the function - the row object, the current data to use in creating the row, and the index of the data. The this object is the table.
data array empty array The array of objects or arrays to load into the table.
display_errors boolean false Indicates whether to display errors on the DOM.
empty_message string 'NO ENTRIES FOUND.' The message to show when no entries are found.
label boolean true Indicates whether to show the label which contains information about the total possible rows and the number of rows current showing.
limits object object Properties include values, which is an array of integers to use as limits, all, which is a boolean indicating whether to add the option to fetch all data, and default, which is the default limit value to use. To set the all as default, set default to 'all'.
loading_message string 'Loading table ...' The message to show while loading the rows.
log_data boolean false Indicates whether to log the received data in an ajax operation to the console.
log_errors boolean true Indicates whether to log errors encountered to the console.
pagination object object Properties include boolean firstLast, which indicates whether to show the First and Last buttons, boolean nextPrevious, which indicates whether to show the Next and Previous buttons, boolean numbers, which indicates whether to show number buttons, and integer maxButtons, which indicates the maximum number of buttons allowed, including the First, Last, Next and Previous buttons if enabled. This defaults to 12.
searchable boolean true Indicates whether the table should be searchable.

Ajax

Request

When fetching data by ajax, the following variables are sent to the page to aid in fetching the right data.

Variable Value Description
limit integer The number of rows to fetch
order array Keys include column, which is the column to order the data by, and dir, which indicates the direction to order the data. dir values may be either 'asc' or 'desc'.
search array Keys include columns, which is a comma-separated string of columns to search based on searchable option set for each column, and value, which is the string being searched.
start integer The number of row to start fetching the data from

Response

A JSON object response is expected with the following properties.

Property Value Description
data array Array of row objects fetched according to the request
total integer The number of all possible rows to fetch

Errors

Code Message Explanation
T0 Error in connection [responseText] This error is generated if there's an error in connecting to the ajax url provided
T1 Response data must be an object The response data must be an object. See Response.
T2 Response object must contain key <total> with an <integer value> The response object must have property total with an integer value. See Response
T3 Data in <Response> must be an array The data property of the response object must be an array. See Response.
T4 DATA.data[<index>] must be either an array or an object This means the data with the given index is not an array or object and should be either. See Response.
